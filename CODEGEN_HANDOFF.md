# Codegen path: 1-layer + full-model run matrix (handoff for further debugging)

## What was tried

Running `tests/benchmark/test_llms.py::test_gpt_oss_120b_tp_galaxy_batch_size_64
--accuracy-testing` through two paths:

1. **Regular compile path** — standard `pytest`, no codegen environment variable
   set. `tt-mlir` compiles to flatbuffer, runs on Galaxy 4×8.
2. **Codegen path** — `CODEGEN_EXPORT_PATH=…` makes `benchmark_llm_torch_xla`
   switch the XLA backend to `codegen_py`, which emits per-graph Python TTNN
   (`graph_N/main.py`, `tensors/`, `utils.py`, `irs/*.mlir`). A separate
   harness (`tests/benchmark/scripts/run_codegen_decode.py`) then loads the
   emitted graphs and executes them on the same Galaxy 4×8 mesh.

For each path we measured against a matched CPU reference (`.refpt` file
generated by `tests/benchmark/llm_utils/reference_generator.py`):

- **Full-model reference** (36 layers): `gpt-oss-120b.refpt` (pre-existing).
- **1-layer reference**: `gpt-oss-120b-1layer.refpt`, generated on-demand
  after wiring `--num-layers` through reference generation (commit
  `91019efa2`).

When running the codegen path we found two prefill SDPA bugs in the emit:

- `attention_sink` kwarg is dropped from `ttnn.transformer.scaled_dot_product_attention`
  (prefill variant) calls, although the corresponding decode-variant call
  (`scaled_dot_product_attention_decode`) gets it correctly. Patched
  per-layer by `tests/benchmark/scripts/patch_attention_sink.py`, which
  copies each layer's `attention_sink=` expression from the decode `main.py`
  into the prefill `main.py`.
- DRAM allocator fragmentation in `ttnn.slice` (`ttnn_slice_943` in the
  full-model graph_0). Allocator log shows 53.9 MB free across 12 banks but
  largest contiguous block 31.4 MB, request 360 MB.

The codegen step itself runs with `dry_run=True` (the runtime returns
zero-filled output buffers from emit-time without executing on hardware —
see `pjrt_implementation/src/api/so_loaded_executable_instance.cc:128`),
so the codegen-step `--accuracy-testing` always prints `TOP1=0.00%`. That
row is excluded from the table below. Real accuracy comes from running the
emitted graphs through the harness and scoring with
`tests/benchmark/scripts/codegen_accuracy_from_log.py` against a matched
`.refpt`.

## Files referenced

### Scripts (tracked on this branch)

- `tests/benchmark/scripts/run_codegen_decode.py` — harness, loads emitted
  graphs and runs `_main()` on a shared mesh device
- `tests/benchmark/scripts/codegen_accuracy_from_log.py` — log → .refpt
  scorer, prints `Token accuracy: TOP1=XX.XX%, TOP5=YY.YY%`
- `tests/benchmark/scripts/patch_attention_sink.py` — structural patcher
  that adds `attention_sink=<expr>,` to each prefill SDPA call by reading
  the analogous decode call's value
- `tests/benchmark/llm_utils/reference_generator.py` — patched to accept
  `num_layers` override (commit `91019efa2`)
- `tests/benchmark/llm_utils/decode_utils.py:init_accuracy_testing` —
  suffixes `.refpt` filename with `-{N}layer` when num_layers is set
- `tests/benchmark/benchmarks/llm_benchmark.py` — emits
  `AUTORESEARCH_PREFILL_PERPOS_ARGMAX_BATCH0` for apples-to-apples scoring

● Test passes. Here's a summary of the investigation:

  Root Cause

  ttnn.sort produces correct sorted values but incorrect indices. This is a
  known bug in the sort LLK (Low-Level Kernel) hardware intrinsics
  (topk_local_sort, topk_merge) — tracked as
  https://github.com/tenstorrent/tt-metal/issues/20625. The stable parameter on
  the sort op is not yet functional (see sort_single_row_single_core.cpp:82-85).

  Evidence:

  ┌──────────────────────────────────────┬───────────────┐
  │                 Test                 │      PCC      │
  ├──────────────────────────────────────┼───────────────┤
  │ Sort VALUES (bf16)                   │ PASS (≥0.999) │
  ├──────────────────────────────────────┼───────────────┤
  │ Sort INDICES (bf16)                  │ 0.989         │
  ├──────────────────────────────────────┼───────────────┤
  │ Sort INDICES (f32)                   │ 0.995         │
  ├──────────────────────────────────────┼───────────────┤
  │ Sort INDICES (unique well-separated) │ PASS          │
  ├──────────────────────────────────────┼───────────────┤
  │ Gather(original, sort_indices)       │ 0.246         │
  └──────────────────────────────────────┴───────────────┘

  When values are close together, the sort correctly orders values but loses
  track of which original position each value came from.

  Impact on DeepSeek Indexer

  The indexer uses topk().indices → scatter to create a sparse attention mask.
  With wrong indices, the scatter masks random positions instead of the correct
  top-k, causing attention to attend to wrong tokens → PCC=0.418.

  Workaround Applied

  Threshold-based masking — uses sort values (which are correct) instead of
  indices:

  # Instead of: scatter(-inf tensor, topk_indices, 0)
  # Use: where(score >= kth_value, 0, -inf)
  threshold = index_score.topk(k, dim=-1)[0][..., -1:]
  index_mask = torch.where(score >= threshold, 0.0, float("-inf"))

  This avoids the broken sort indices entirely. The topk().values are correct
  because sort values are correct. The comparison + where ops work correctly on
  TT.

  Changes Made

  1. modified_model.py: Indexer returns raw scores; MLA uses threshold-based
  masking instead of topk_indices → scatter in both prefill and decode paths
  2. test_deepseek_v3_2_exp.py: Points to V3.2 checkpoint, maps indexer weight
  keys, includes index config fields
